# Fixes Aplicadas — MRend.js (2026-05-29)

## Contexto
O utilizador reportou que valores editados desapareciam após refrescar a página. O documento `CONTEXTO-SESSAO-2026-05-29.md` identificava a causa como um problema na persistência de edições ao refrescar quando os dados são re-fetchados do servidor.

---

## Bugs Identificados e Corrigidos

### Bug 1 — CRÍTICO: `_applyPendingEdits` procurava pelo campo errado no Dexie
**Ficheiro:** `MRend.js`  
**Linhas:** 859-875 (função `_applyPendingEdits`)  
**Problema:**
```js
// ANTES (bugado):
.where("cellId").equals(cellId)
```
Estava a hardcoding procurar pelo campo `"cellId"`, mas o Dexie armazena o ID da célula num campo configurável (`cellIdField`, ex: `"u_reportlstamp"`).

**Solução:**
```js
// DEPOIS (correcto):
var cellIdFieldName = mrendThis.dbTableToMrendObject.extras.cellIdField || "cellId";
var promises = cellIds.map(function (cellId) {
    return mrendThis.db[mrendThis.tableSourceName]
        .where(cellIdFieldName).equals(cellId)
        .modify(edits[cellId])
        .catch(function () { /* registo pode não existir ainda */ });
});
```

**Impacto:**  
Sem esta correção, quando o Dexie é sobrescrito após refresh, `_applyPendingEdits` tentava restaurar as edições mas nunca encontrava os registos (porque procurava pelo campo errado). Resultado: valores editados eram perdidos.

---

### Bug 2 — CRÍTICO: `buildDefaultCelula` hardcodava `sourceKey` como "cellId"
**Ficheiro:** `MRend.js`  
**Linhas:** 3731 (in `buildDefaultCelula`)  
**Problema:**
```js
// ANTES (bugado):
bindData: new BindData({ sourceKey: "cellId", sourceBind: coluna.config.campo }),
```
O `sourceKey` é usado em `updateCellOnDB` (linha 1351) para procurar o registo no Dexie:
```js
.where(updateData.sourceKey).equals(updateData.sourceValue)
```
Se `sourceKey = "cellId"` mas o campo real é `"u_reportlstamp"`, as atualizações não funcionam.

**Solução:**
```js
// DEPOIS (correcto):
var cellIdFieldName = mrendThis.dbTableToMrendObject.extras.cellIdField || "cellId";
var cellObjectConfig = new CellObjectConfig(
    {
        bindData: new BindData({ sourceKey: cellIdFieldName, sourceBind: coluna.config.campo }),
        // ... resto dos campos
    });
```

**Impacto:**  
Sem esta correção, `updateCellOnDB` falhava a atualizar o Dexie quando uma célula era editada (porque procurava pelo campo errado). Isto significava que mesmo com `_savePendingEdit` a guardar em localStorage, o Dexie nunca era atualizado.

## Fluxo Corrigido

### Antes (bugado):
```
Edição de célula
  → syncChangesToDB
      → updateCellOnDB(...where("cellId")...)  ✗ Falha (campo errado)
      → _savePendingEdit(cellId, campo, valor)  ✓ Guarda em localStorage
      
Refresh
  → Stamps diferentes → re-fetch do servidor
  → deleteAllRecords → Dexie vazio
  → addBulkData → Dexie com dados originais
  → _applyPendingEdits(...where("cellId")...)  ✗ Falha (não encontra registos)
  → Resultado: Valor original, edição perdida ✗
```

### Depois (correcto):
```
Edição de célula
  → syncChangesToDB
      → updateCellOnDB(...where("u_reportlstamp")...)  ✓ Sucesso
      → _savePendingEdit(cellId, campo, valor)  ✓ Guarda em localStorage
      
Refresh
  → Stamps diferentes → re-fetch do servidor
  → deleteAllRecords → Dexie vazio
  → addBulkData → Dexie com dados originais
  → _applyPendingEdits(...where("u_reportlstamp")...)  ✓ Sucesso
  → Resultado: Valor editado restaurado ✓
```
